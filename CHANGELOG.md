## 2026-06-24

- **Fix App Store upload rejection (HealthKit, error 90683)**: re-added `NSHealthUpdateUsageDescription` to `Info.plist` (Base) + `en.lproj`/`zh-Hans.lproj` `InfoPlist.strings`. Apple's ITMS upload validator requires **both** HealthKit purpose strings (`NSHealthShareUsageDescription` *and* `NSHealthUpdateUsageDescription`) whenever the HealthKit entitlement is present — regardless of whether the app writes. The audit had removed the Update string ("read-only ⇒ no write string"), which is correct app-behavior reasoning but fails Apple's static check, so `Distribute → Upload` was rejected with code 90683. The re-added string honestly states Rede never writes to Health and only reads body weight for on-device display; it is never displayed (the app never requests write authorization), it only satisfies the validator. **Entitlement untouched** — no signing change. Verified by unpacking the built `Rede.app` (both keys present; version 0.2.0 (7), MinimumOSVersion 17.0 unchanged) + quality gate PASS. Re-archive 0.2.0 (7) and re-upload.
