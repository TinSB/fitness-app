# IronPath UI 视觉验收清单

本清单用于每次发布前手动检查 Web/PWA 的产品级版式。目标是确认页面职责清楚、桌面端不浪费空间、手机端训练操作高效。

## 手机端

- [ ] 顶部没有重复 safe area 或大块空白。
- [ ] 今日页顶部结构为状态栏 / 紧凑品牌区 / 16-24px 内容间距，不需要手动上滑才正常。
- [ ] 底部导航不遮挡页面内容和主按钮。
- [ ] 页面不出现横向滚动。
- [ ] 关键按钮高度足够触摸，文字清楚。
- [ ] 空状态轻量，并给出下一步操作。
- [ ] 危险操作使用红色样式并有确认。
- [ ] Drawer / BottomSheet / 训练详情顶部关闭按钮避开 iPhone 状态栏。

## 桌面端

- [ ] 左侧导航固定显示，主内容使用 1180-1280px 宽度。
- [ ] 不出现中间窄列加右侧大片空白。
- [ ] 页面支持主内容 + 右侧辅助栏。
- [ ] 卡片、标题、按钮、标签视觉一致。
- [ ] 信息密度适中，右侧辅助栏卡片紧凑。

## 今日

- [ ] 首屏只回答“今天是否训练、练什么、从哪里开始”。
- [ ] 主 CTA 明显：开始训练 / 继续训练 / 查看本次训练。
- [ ] 已完成状态显示“今日训练已完成”，不显示“开始训练”为主按钮。
- [ ] 下次训练显示为“下次建议”，不误导成今日必须继续训练。
- [ ] 完成腿 A 后，下次建议按模板轮转，不应静默继续腿 A，除非有明确原因。
- [ ] 准备度、预计时长、训练模式、动作数不重复出现。
- [ ] 完整动作安排默认折叠。

## 专注训练

- [ ] 一屏内可看到当前动作、阶段、组进度、建议和实际记录。
- [ ] 完成一组是最明显按钮。
- [ ] 重量 / 次数快捷调整不拥挤。
- [ ] 底部操作区始终可见复制上组 / 标记不适 / 替代动作。
- [ ] 替代动作使用统一 bottom sheet；无替代项时显示 toast，不静默失败。
- [ ] Toast 反馈统一，不重复堆叠。
- [ ] 默认不显示长解释、证据来源或 PR 说明。

## 记录

- [ ] 默认打开日历。
- [ ] 桌面端日历为核心区域，右侧显示选中日期详情。
- [ ] 手机端使用日历 / 历史 / 统计 / PR / 数据短标签。
- [ ] 无历史时不显示大量 0 统计。
- [ ] 历史详情用 Drawer 或明确详情区。
- [ ] 删除、标记测试、恢复正常都有确认或明确反馈。
- [ ] 历史详情可进入“修正记录”，保存前提示 PR、e1RM、有效组和统计会重新计算。
- [ ] 训练详情顶部指标与动作记录一致：有正式组时完成组数和总量不能为 0。

## 计划

- [ ] 当前计划状态置顶且清楚。
- [ ] 桌面端主内容按当前计划 / 本周安排 / 待处理建议 / 调整草案组织，右侧栏只显示摘要。
- [ ] 手机端按当前计划、本周安排、待处理建议、调整草案顺序展示。
- [ ] 不显示历史训练详情、日历或 PR 大面板。
- [ ] 回滚等危险操作使用 danger 样式。

## 我的

- [ ] 像设置中心，而不是训练执行页。
- [ ] 筛查、单位、健康数据导入、数据管理入口清楚。
- [ ] PWA / 本地数据说明短且可理解。
- [ ] 备份 / 恢复不藏在深层页面。
