# `ios/ParityFixtures/parity/`

This directory contains frozen JSON fixtures consumed by Swift package tests.

These files are historical parity assets. They are kept because the native Swift engines still use them as deterministic acceptance inputs and expected outputs. They are not a live generator, not an alternate source of truth, and not permission to restore the removed Web runtime.

## Current Rules

- Swift tests may read `inputs/` and `golden/`.
- Golden files must not be edited by hand unless the corresponding Swift test contract is intentionally changed and revalidated.
- The old generator and web source tree are no longer part of this repository.
- Do not add Node, Vite, browser, package-manager, or web-build steps to refresh these fixtures.
- New native fixtures should be authored or generated by a Swift-native, explicitly approved test tool.

## Layout

```text
ios/ParityFixtures/parity/
├── inputs/
└── golden/
```

`inputs/` files may still contain provenance metadata such as historical commit ids or generator names. Treat those as frozen audit metadata only. They do not describe current tooling.

## Validation

Run the Swift package tests that consume the touched fixture:

```bash
cd ios/packages/IronPathTrainingDecision
swift test
```

For a broad pass:

```bash
for package in ios/packages/*; do
  if [ -f "$package/Package.swift" ]; then
    (cd "$package" && swift test) || exit 1
  fi
done
```
