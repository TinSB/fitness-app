# Larder — 图标 / 元素 / 文字 / 语言设计指南

> 2026-06-04 · 基于美国消费者偏好调研(产品设计团队 + 前端 UI 工程师联合,均带来源)
> 适用:与 IronPath 联动的 iOS 饮食 App「Larder」(暂定名)· 暖色 / 精品 / ED 友好 / 低摩擦 · en-US 主、zh 次

**一句话**:美国健康类用户当下偏好**克制渐变 + 单焦点符号的 App 图标、带文字标签的填充式应用内图标、留白足的卡片、人文/圆体无衬线 + sentence case、"友好但专业、6–8 年级"的文案**——而这套主流偏好,对 ED 友好饮食 App 恰好与"去数字化、非评判、低摩擦"同向。关键动作不是图标长什么样,而是**把卡路里/体重从视觉中心移走**。

---

## 1. App 图标(App Store / 主屏)

**美国偏好(数据)**:约 65% 下载来自搜索,图标是头号转化杠杆;**简单高对比图标转化≈复杂图标 2 倍**(某 A/B:+22% 自然安装);2026 主导 = **同类色渐变背景 + 干净单焦点符号**(紫蓝/橙粉/青绿,忌互补高对比小尺寸发脏);健康/正念品类靠**情绪色 + 流体形状**取胜(Calm 逻辑);文字图标走低;**iOS 18+ 必做 Light/Dark/Tinted 三套**,苹果自家图标在 dark/tinted 里用渐变+alpha 保层次,第三方丢扁平字形会"失真"。

**决策**:
- 主图标走**暖色同类渐变(蜜橙/赭)+ 亮面绿能量球(单焦点)**;不堆高饱和"食欲色"红/亮黄(违精品 + ED 回避食欲刺激)。
- **现在就出三套**(Light/Dark/Tinted),手工调,不交给 Xcode 自动转;tinted 用灰阶 alpha 渐变保住球的体积,dark 去背景。
- **图标内不放文字、不放食物照片**。能量球是"felt, not literal"的非具象焦点,契合品类规律。
- 上线后用 Apple **Product Page Optimization** A/B 测图标。
- 3 个具体方向见 §6。

## 2. 应用内图标

**美国偏好(数据)**:**带文字标签的图标识别正确率 88%,无标签 60%,App 独有图标无标签仅 34%**(NN/g);"真正通用"图标极少;iOS 约定 **Tab Bar 用填充、导航/行内用线性**;SF Symbols 是用户最熟悉的图标语言(6,900+,与系统字对齐),自定义图标越贴其度量越易读;"5 秒规则"——想不出图标就别用图标。

**决策**:
- **应用内主干用 SF Symbols 7**(系统一致、零授权、自动 Dynamic Type/字重对齐、`symbolEffect` 自带 Reduce Motion);**占位的 Tabler 上线前必换**(度量不对齐、不随字号缩放)。
- **食材大类字形自绘成 `.symbolset`**(用 SF Symbols app 的变量模板对齐 cap-height/baseline、保留图层命名、outline+fill 双变体),这样和系统符号一样吃字号/字重/动效/基线。资产命名 `ingredient.protein` / `ingredient.veg`…
- **变体工程**:Tab Bar 写 base name 让系统自动填充(不手写 `.fill`);列表行内用 outline,选中切 fill(`.symbolVariant`)。
- 线性 + 圆角端点(呼应圆体字 + 暖调),统一线重(≈SF Symbols Regular/Medium 光学重量)。
- **任何可点导航图标必配可见文字**(88% vs 60% 是留存与误操作的直接成本);上线前做 5 人识别测试(尤其"坚果 vs 豆类""油脂 vs 调味")。

## 2A. 食材视觉：照片 vs 字形 —— 按"场景"分层，而非按"品类"

> 2026-06-04 · 产品设计团队 + 前端 UI 工程师双线调研结论(均带来源)

**为何不是全照片**:任何高保真食物影像在 ED 临床里本就是"暴露刺激物",还会升高饥饿激素——与"回避食欲刺激"定位相悖;密集小尺寸列表里写实照片是视觉噪声,反比字形慢识别(NN/g、Boxes&Arrows);成百上千 SKU 拍不全、跨图打光不齐会摧毁精品感;AI 生食材落在"almost-real"恐怖谷区(Instacart 已翻车);Widget 30MB 内存预算基本不能放照片。
**为何不是全字形**:纯字形在"精品 / 生鲜可信 / 无歧义"上弱,是产品方反复纠结的真实短板。

**决策(发散→量化对比后,"按场景分层"4.18 > "按品类混合"3.44,且为所有含照片方案里最高)**:把照片从"高频默认浏览态"挪到"低频主动详情态"。

- **密集列表 / 库存网格 / 食材选择器 / Widget → 全字形 + 文字标签,零照片**。日常高频扫视区:字形识别更快、零情绪唤起、零长尾覆盖问题、与玻璃 / 圆体语言同源。
- **详情页 / 今日推荐主食材(英雄位)→ 仅高频 ~40–60 个食材用白底抠图真实照片**。平光、**去诱食打光**(无油亮 / 蒸汽 / 酱汁),装进统一暖色圆角卡;深色模式用语义垫色,不裸露白底。
- **生肉 / 生鱼 / 生禽 → 即便英雄位也不用裸生肉照**(临床反胃 + 最违背 ED),用字形或分切 / 熟成的克制图。
- **包装 / 条码食品 → 走条码图源在线展示**(Open Food Facts;CC-BY-SA 许可需法务确认),拉取失败回落字形。
- **长尾 / 自定义 → 字形 + 品类底色兜底**,允许用户自拍走抠图管线入库。
- **全局"字形模式"开关**:一键把所有照片切字形,给 ED 敏感用户零影像安全态。
- **一致性外壳**:照片与字形共享同一圆角卡 / 尺寸网格 / 内边距 / 品类底色——切换时容器不变、只换内容物,感知为"统一图标系统"而非"有的有照片有的没有"。

**排期(运营视角):先不投拍摄**。用字形上线验证 PMF;照片作为 v1.x 精品化升级,先用授权图库做 5–10 个高频食材试点,确认转化 / 留存提升后再投核心库(自拍摄 150–300 张约 $15k–40k)。

**主要来源**:NN/g(图标可用性)· Boxes&Arrows(小尺寸识别)· Frontiers / PMC(生肉厌恶、食物图与饥饿激素、ED 暴露刺激)· Oxford 2024 & Appetite 2025(AI 食物图恐怖谷)· Instacart Tech Blog / AIAAIC(AI 食材图翻车)· Open Food Facts(条码图源 / 许可)· Apple WidgetKit(30MB 预算)· Unsplash License。

## 3. 视觉元素

**美国偏好(数据)**:3 秒内凭视觉定品牌;健康类信任来自**科学/认证/权威背书**(91% 看重"有效性");照片对电商型转化有量级影响,但**饮食 App 内的食物特写在 ED 语境是风险**——临床医生设计的 Recovery Record 刻意不挂卡路里库、以"非评判"为第一原则。

**决策**:
- **卡片 + 留白 + 线性食材字形为主,慎用诱惑性食物特写**(满足精品 + ED 安全)。若用照片走"原料、自然光、去造型",绝不用 before/after、理想身材。
- 信任靠**可见的权威背书**(USDA/IOM/FDA 来源做低调标签,与美标策略一致)> 视觉炫技。
- **绿能量球是全界面唯一"高光泽"焦点**,其余哑光低刺激;暖色 + 大留白即精品。

## 4. 文字 / 排版

**美国偏好(数据)**:**sentence case 更友好更易读**(Material 3 全量 sentence case;文字变长时尤其);**人文/圆体无衬线 = 温暖信任**(几何体显冷峻;wellness 常用 Nunito Sans / Museo Sans);数字对齐用 **tabular-nums**、表格右对齐、大数用 K/M 单位。

**决策**:
- **全 UI sentence case**(按钮/标题/列表/空状态);品牌名 Larder/IronPath 保留首字母大写。
- **字体用 SF Pro Rounded(标题/数字)+ SF Pro(正文)**——圆体暖、克制、**零授权 + 自动 Dynamic Type + 自动中文回退(PingFang)**;不引入自定义西文字体(会带来自管缩放/体积/中文另配/中英度量失衡)。
- 营养数字一律 `.monospacedDigit()`(防跳动);但**字号/色弱化,移出视觉中心**。
- 语义字号 + `@ScaledMetric` 图标随字号长;AX5 不截断(宁换行不缩字);并排"图标+文字"在 AX 档切纵向。

## 5. 语言 / 文案

**美国偏好(数据)**:多数健康 App 描述是 **11–15 年级**阅读难度,仅 6% 达推荐的 3–7 年级,**可读性预测安装量**;"友好 = 尊重 + 清晰,不是耍宝";**emoji 在严肃健康语境慎用**(含义模糊、跨平台/年龄理解不一);ED 友好语气共识:**中性、不给食物贴好/坏标签、不施内疚、反思式问句、即使当天紊乱也给鼓励**,警惕"换皮节食"。

**决策**(与 MEMORY「文案专业简洁、去 AI 腔」一致):
- 目标**6–8 年级**(Hemingway 校验);短句、主动语态、具体名词。
- 语气标尺:_"像一个懂行、不评判的朋友,帮你用现有食材做顿饭。"_ 温暖、专业,不卖萌、不施压、不过度安抚。
- **功能 UI / 营养反馈默认不用 emoji**;情绪记录可用一组固定、含义明确的符号。
- 去评判清单:不写"超标/超出/吃多了"、不贴"健康/不健康/cheat meal"、不设"预算/上限/达标"框架;空/完成态用鼓励;数据不全不做惩罚性提醒;少破折号、少营销感叹号。
- **卡路里/体重默认弱化、可选**;不每日称重、不设热量上限、不做大字号卡路里仪表盘(临床列为触发源)——与低摩擦定位天然一致。
- 内置克制的"获取支持"入口,指向**美国本地、现行有效**的 ED 专业资源(注意 NEDA 求助热线已停用,勿引用)。

## 6. App 图标 — 3 个方向(细到可作画)

共享:1024 圆角方;同出 Light/Dark/Tinted;前景单焦点、无文字、无食物照;暖色精品低刺激。

- **A ·「暖光储藏室之球」(主推)**:背景暖金→蜜橙同类色对角渐变,居中亮面绿能量球(左上高光、底部暖反光),球占 ~55–60%。暖=家/储藏归属,绿=新鲜不刺激。Dark 去背景留高光;Tinted 灰阶 alpha 保体积。转化最稳。
- **B ·「储藏室拱门/罐口 + 球」(品牌叙事)**:奶油哑光底 + 极简暖赭线性"罐口/拱门"轮廓(呼应 Larder 词源),内嵌小亮绿球——"打开储藏室的一抹生机"。独特可记,线性最易适配三态;抽象度偏高,靠名+副标补足。
- **C ·「极简能量球字标」(SF 原生/对照组)**:暖金→奶油轻渐变,只有一颗更精致的发光绿球(次表面散射感)。极简高级、三态最省力;品类传达最弱,靠 ASO 文案补。

**推荐**:A 主投、B 作品牌叙事储备、C 作 A/B 测试极简对照。

## 7. 本地化与无障碍(工程)

- **String Catalog(.xcstrings)** 外置全文案、加 comment;单位/日期/数字用 `Measurement`/`.formatted()` 按 locale(美标=数据来源,格式=按 locale,两件事);中文段落行高 ~1.7×;图标不内嵌文字;上线前跑伪本地化;`.leading/.trailing` 预留 RTL。
- 每图标 VoiceOver label(装饰性 `.accessibilityHidden`);**图标必配可见文字**;对比 ≥AA 且信息不只靠色;触控 ≥44pt;动效优先 `symbolEffect`(自动尊重 Reduce Motion)。
