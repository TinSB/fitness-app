## Faithfulness notes

- **Reuse, don't re-port.** The truly-shared set helpers (`number` / `setWeightKg` /
  `isCompletedSet` / `completedSets`) are reused verbatim from the iOS-17e-1 `E1RMEngine`.
- **Genuinely-distinct helpers ported fresh.** The analytics `safeDate` normalises EVERY
  date-prefixed value to **noon UTC** (E1RMEngine's parses bare dates as midnight and does
  not prefix-normalise full ISO), and the analytics `isAnalyticsSession` ALSO requires
  `completed !== false` (E1RMEngine's checks only `dataFlag`). These are different
  functions, so porting them fresh is correct — not duplication.
- **Zero `: Date`.** "今天" is the injected `options.nowIso`; the TS `|| new Date()` /
  `?? Date.now()` wall-clock fallbacks are intentionally NOT reproduced (the §11 clean
  input always supplies `nowIso`). All UTC week/month math is **integer civil-calendar
  arithmetic** (Howard Hinnant days⇄civil), never `Date()` / `Calendar`.
- **Map insertion order.** `recentPRDeltaEngine`'s observation grouping and
  `weeklyMuscleBalanceEngine`'s `allMuscles` reproduce JS `Map` first-encounter order via
  an ordered (order-array + dict) structure; the JS-stable sorts are reproduced with
  Swift's stable `sorted(by:)`. Muscle-balance fixtures use only focus muscles, so
  `muscleContribution` key order (alphabetised on JSON ingest) is result-irrelevant
  (per-muscle accumulation is commutative; `allMuscles` uses focus order).
- **JS number formatting.** `Number(x.toFixed(n))` → `AnalyticsSupport.roundToFixed`;
  `Math.round` → `jsMathRound` (`floor(x + 0.5)`). Fixtures use non-tie decimals so the
  ported rounding matches JS bit-for-bit (the compute-assert pins exact Double `==`).

## Parity

3 NEW goldens under `tests/fixtures/parity/golden/{training-streak,recent-pr-delta,weekly-muscle-balance}/`,
each a `cases` array (one case per branch). GENERATED by
`scripts/generate-parity-goldens.mjs` (→ `generateTrainingStreak` / `generateRecentPRDelta`
/ `generateWeeklyMuscleBalance`), **never hand-edited (§22)**. The Swift
`*ParityTests` decode each case's echoed `history` + `options`, re-run the ported engine,
and assert the produced result == the golden case `result` item-by-item.

Coverage:

- **training-streak** — no-history / active-week-streak / carry-last-week / broken-streak /
  filtering (test+excluded+completed:false excluded) + month run.
- **recent-pr-delta** — up/down/flat directions / new + skip (out-of-window only) /
  sort-new-first + `limit` slice / window cutoff+now boundary + `pickBest` reps tie-break.
- **weekly-muscle-balance** — no-data-this-week / balanced (score 100) / imbalanced
  (overworked + underworked) / `muscleContribution` + secondary-primary allocation +
  week & dataFlag filtering.
