# Larder · Clay 食材图标 — Blender 实现 Brief

> 2026-06-04 · 给 Blender 连接器 / Claude Code 的实现基准。配套视觉稿见 `larder-clay-foodset.html`(色板与 7 条规则即本 brief 的视觉来源)。
> 工具:Blender 4.2+ · Claude Desktop「Blender 连接器」(自然语言驱动 Python API,内置 Hyper3D Rodin/Hunyuan3D 文生 3D)。

---

## 0. 目标 / 范围 / 验收

**目标**:产出一套**风格统一的暖哑 claymorphism 生食材** PNG(透明背景),首批 12 个高频食材,可扩展到核心 80–150。最终进 iOS Asset Catalog。

**范围内**:母场景(材质/灯光/相机/渲染/导出)、12 食材建模配方、批量渲染与后处理、命名规范、垂直切片验证。
**范围外**:iOS 端 SwiftUI 集成(已有架构,见 clay 可行性结论)、长尾兜底字形(已定 `.symbolset`)、Widget 策略(已定回落字形)。

**验收标准**(每条都要满足):
1. 12 个 PNG,**1024×1024,透明背景**,主体居中、占画面 **~70%**。
2. **同一相机 / 同一灯光 / 同一材质**,12 张并排肉眼**无风格漂移**(光向一致、色温一致、视觉重量一致)。
3. **暖哑**:无镜面高光、无玻璃感、无霓虹(与"渐变贴纸"分水岭)。
4. **生鸡胸非临床**:圆润哑光暖肉粉,不血色不写实。
5. 落影**暖色**(非灰黑),柔和接触影。
6. 每个导出 @2x(512)/@3x(768) 或保留 1024 交 iOS 降采样;命名 `ingredient.<name>.png`。
7. 深色无白边(导出直 alpha / defringe)。

## 1. 母场景规范(Master scene,搭一次锁死)

> 这是统一的命门——所有食材复用同一套 World / Camera / Light / Material,只换模型与基色。

### 渲染
- 引擎:出稿用 **EEVEE Next**(快);定稿用 **Cycles** 128–256 samples + Denoise。
- 输出:**1024×1024**;`Render → Film → Transparent` **勾选**。
- 色彩管理:`View Transform = Standard`(保色准,**不要 Filmic**),所有食材统一,严禁逐图改。

### 相机(锁定,全食材复用)
- **Orthographic**(首选,零透视畸变)或 50–85mm 长焦弱透视远距拍。
- 俯角 **~20–25°**(略俯,见顶面体积)。
- `Ortho Scale` 调到主体填充画面 **~70%**;位置/角度锁死。

### 灯光(三点,暖调;对应 SVG 的"左上高光")
| 灯 | 位置 | 色温 | 强度 | 作用 |
|---|---|---|---|---|
| Key 主光 | **左上 45°** | 暖白 ~5000–5200K | 基准(如 area 0.3m / 80W) | 定义体积,亮面在左上 |
| Fill 补光 | 右前下 | ~5600K | Key 的 ~25–30% | 提暗部,不抢主光 |
| Rim 轮廓 | 后上 | 冷一点 | 弱 | 勾边,与白底分离 |
| World 环境 | — | 暖 | 极弱 0.03–0.05 | 保底,不产生方向性 |

### Clay 材质(统一材质球,Principled BSDF · Blender 4.2)
- `Base Color` = 食材**基色**(见 §2 色板的"基")。
- `Roughness` = **0.87**(哑光)。
- `Metallic` = 0;`Specular` 低(~0.2);`Coat Weight` = 0;`Sheen Weight` = 0;`Emission` = 0。
- `Subsurface Weight`:**有机体** 0.05–0.12(番茄/鸡胸/牛油果肉/叶菜),**干货** 0(杏仁/米/燕麦粒);`Subsurface Radius` 取比基色略暖红。
- **做成一个可复用 Material**,每个食材实例只改 `Base Color` 与 `Subsurface`。
- **不靠材质镜面做高光**——高光由 Key 主光在左上自然形成(哑光体积感的关键)。

### 接触阴影
- Cycles:不可见地面 `Shadow Catcher`;或图标下方烘一片**暖色软椭圆影**(`rgba` 近 120,60,30)。落影绝不灰黑。

### 导出
- PNG **RGBA**,sRGB,**直 alpha**(straight,非 premultiplied)避免深色白边;必要时后期 defringe/decontaminate 边缘。

## 2. 12 食材建模配方(基础几何为主 = 风格最统一)

> 路线:**程序化/手工基础形为主**(量化推荐 C),只在复杂有机体用 metaball/Rodin 起形。**不要让 Rodin 出写实网格**——会破坏统一,且要重做材质。

| # | 食材 | 基础几何 + 手法 | 基色(亮 / 基 / 暗) | Subsurface |
|---|---|---|---|---|
| 1 | Tomato 番茄 | UV 球 → Scale Z 0.85 略扁 → 顶底轻内凹;蒂=5 瓣小星锥 | #f3937c / **#d75a3e** / #b8442c;蒂 #6f9e4a | 0.10 |
| 2 | Eggs 鸡蛋 | UV 球 → Lattice 拉成蛋形(一端略尖) | 壳暖白 **#efe3cd** | 0.06 |
| 3 | **Chicken 生鸡胸** | 球/立方 → Sculpt 或 metaball 捏**圆润肾形肉块**,表面光滑 + 一道极浅 crease | 暖肉粉 **#e6ad94** | 0.12(肉感不血色) |
| 4 | Oats 燕麦 | 碗=半球切口+Solidify;碗内椭粒堆(particle/手摆) | 碗 #d2a566;燕麦 **#e0c98f** | 0 |
| 5 | Spinach 菠菜 | 3–4 片平面 → Subdivide → 比例编辑捏叶形+轻卷;叶脉浅 crease | **#5b9e4e** | 0.08 |
| 6 | Avocado 牛油果 | 梨形球切半 → 内挖浅碗;核=小球 | 皮 #3f6e30 / 肉 **#c2d36a** / 核 #7a4f28 | 肉 0.10 |
| 7 | Banana 香蕉 | 曲线+圆截面 bevel,或 cylinder+Simple Deform(Bend);三棱轻压 | #f5dd86 / **#e8c24f** / #c79a36 | 0.04 |
| 8 | Broccoli 西兰花 | 花球=metaball 小球簇(或 ico 球群);茎=锥柱 | 花 **#3f7a30** / 茎 #b7c98a | 0.06 |
| 9 | Salmon 三文鱼 | 扇形块(cube→倒角→比例捏);白纹=几条浅色细条 | **#e8896b**;纹 #f9e6da | 0.10 |
| 10 | Rice 米 | 碗同燕麦(换暖灰陶);米堆=白小粒 | 碗 #b0a07e;米 **#f3efe6** | 0 |
| 11 | Greek yogurt 希腊酸奶 | 杯=圆台(上大下小);顶一抹酸奶卷 | **#f3efe6** | 0.03 |
| 12 | Almond 杏仁 | 变形椭球(一端尖)→ 中线浅 crease | #ecc896 / **#d8a86a** / #b8854a | 0 |

**每个食材统一流程**:取/捏形 → 实例化母场景 clay 材质 → 改 Base Color/Subsurface → 摆进相机框 70% → 渲染。
**复杂三个(鸡胸/西兰花/三文鱼)** 若手捏慢,可用 Hyper3D Rodin 文生网格**起形**,但落地后**一律换成母场景 clay 材质**,绝不保留 Rodin 的写实材质。

## 3. 批量渲染与后处理(伪流程)

```
母场景.blend(锁灯光/相机/渲染/材质) ── 模板
      │
      ├─ 每个食材一个 Collection(或单独 .blend,link 母场景 World+Camera+Light)
      │
      └─ 批处理(脚本):
           for 食材 in 列表:
             隐藏其他 Collection → 设输出名 ingredient.<name> →
             render(1024² 透明) → 存 PNG
      │
      └─ 后处理(脚本/工具):
           pngquant/oxipng 无损压缩 →
           生成 @3x(768)/@2x(512) 或保留 1024 →
           defringe 去白边(深色 halo) →
           QC:12 张并排 + 平均 hue 落暖区检测
      │
      └─ 交付:ingredient.<name>@2x/@3x.png → iOS Asset Catalog
```

## 4. 交给 Claude Code 的脚本任务(实现 brief,非本文件交付代码)

> 连接器适合交互搭场景/捏单体;**可复用的批处理脚本交 Claude Code**。每个任务给目标+验收,不在此写运行代码。

1. **母场景生成器**:按 §1 参数化建灯光 rig(Key/Fill/Rim)+ 正交相机 + clay 材质节点;输出 `master.blend`。验收:渲染一个测试球=暖哑、亮面左上、透明底。
2. **程序化几何生成器**:按 §2 为可参数化食材(番茄/蛋/杏仁/香蕉/牛油果/碗类)生成 primitive+修改器函数,入参=尺寸/压扁/弯曲度。验收:跑出 6 个基础形,比例正确。
3. **批量渲染器**:遍历食材 Collection → 渲染 → 命名导出 1024² 透明 PNG。验收:一键出 12 张。
4. **后处理**:压缩 + 生成 @2x/@3x + defringe + hue QC 报告。验收:导出符合 §0 验收 1–7。

---

## 5. 先做垂直切片(强制,降风险)

**先只做番茄走通全链**:母场景 → 番茄基础形 → 套 clay 材质 → 渲染透明 PNG → 放进 `Kitchen` 列表行 + 详情英雄位看观感。
**通过验收(暖哑、左上光、占 70%、深色无白边)再批量**——避免一次性建 12 个后发现风格/导出问题返工。切片几小时,挡住几天返工。

---

## 6. 注意事项

- **Rodin 何时用**:仅复杂有机体起形,且必换母场景材质;简单食材**别用** Rodin(写实网格难统一、是弯路)。
- **深色 halo**:导出直 alpha;iOS 端图标统一进圆角卡容器(已定方案 A),双保险。
- **一致性铁律**:相机、灯光、材质、色彩管理**全程不改**;新食材只改基色与形,放进已有 12 张并排目测过审再入库。
- **角色分工**:交互搭场景/调材质/捏单体 → 连接器(连上后可代驱动);批处理/后处理脚本 → Claude Code(凭 §4)。
