## N3 · 今日页图形语言：周分段条 + 体量 sparkline（后做，独立 commit）

### N3a 状态行周分段条

**现状**（TodayTabView ~397-407 `contextLine`）：状态点 + 状态词 + 右侧 `signalLineText`（「近 7 天 N 次 · 计划 M」——**滚动 7 天口径**）。

**改动**：
1. **新纯函数**（放 RedeLocalSnapshot，紧挨 ContinuityCalendar 的派生风格）：输入 = history 的 dateISO 集合 + todayISO，输出 = 本**日历周**（周一始，ISO 周，与 ContinuityCalendar/周量图同口径）7 天的状态数组：`trained / today(未练) / past(空) / future`。TDD：先写测试（跨周边界、今天=周一、今天已练、空历史四个 case 起步）。
2. **UI**：`contextLine` 右侧改为「7 格分段条 + 计数文字」。格子几何 = 月历 dayCell 的缩微版语法：trained = redeEmber 实心小方/圆，today 未练 = redeT4 描边，past 空 = redeEtch 刻点，future = redeEtch 刻点（更低调或同 past——实施时试两版取干净者）。尺寸目标 ~6-8pt 格、间距 3-4pt，整条 ≤ 70pt 宽，不与左侧状态词挤压（XS/XXL 档验证）。
3. **口径一致红线（本批最大设计雷）**：分段条是日历周，旁边文字**必须同口径**——把状态行右侧文字改为日历周计数（如「本周 3 次」，L10n 新串 zh/en），`signalLine` 的滚动 7 天串不再在此处展示（引擎信号照旧，只动展示）。一屏之内禁止两种周口径并存。gapDays 语境（「距上次 N 天」）若现文案含此信息，保留语义的归属由你判断，但不得引入第二行常驻文字。
4. **a11y**：整条分段条 + 计数合成一个元素，读法类「本周已练 3 天」（具体串双语，格子本身 a11y 隐藏）。

**数据接线**：today 侧现有 model/verdict.signals 没有「本周哪几天」——从 canonical history 日期派生（loadCompletedDigest 已示范从 ProgressModel/SessionStore 取历史的两条路，选最轻的一条；禁止在主线程重跑引擎——批次 F 审查 M-1 教训）。
