## 总控指令（给执行会话）

你要按顺序完成下面 6 个任务（Task 1 → 2 → 2b → 4 → 5 → 3，Task 6/7 为 follow-up 可选）。纪律：

- 每个任务 = 独立分支 + 独立 PR + 单独验收，命名 `codex/2026-MM-DD-<任务名>`；从最新 `main` 起步，开工前 `git fetch --prune` 并确认工作区干净（有并行会话，提交用 pathspec 只带自己的文件）。
- 修 bug 走标准闭环：先写**失败测试**（红）→ 最小修复（绿）→ 回归。禁止顺手重构、禁止夹带无关改动。
- 每个 PR 合并前必过：`.claude/quality-gate.cmd` 全绿 + 独立 code-reviewer 子 agent 审 diff + 模拟器实拍验收截图。
- 文案改动必须中英成对过文案基线八问（docs/REDE_PRODUCT_COPY_BASELINE.md）：UI 短语无句号、禁词表零出现、两语各自母语表达不逐句机翻。
- L10n 串改动同步更新 `RedeL10n` 包内 golden/禁词测试。
- 全部任务**不碰** AppData schema、不碰训练历史写路径、不碰引擎轮转逻辑（越界即停下报告）。
- auto-merge 仅限门禁全绿 + 审查无 MAJOR + 纯 UI/文案/主题范围；任一存疑就只开 PR 等 owner。
- 每个 PR 合并后追加 DEV_LOG 条目（给不读代码的 owner：现象/为什么要紧/改了什么/怎么证明）。
- 批次收口：规格写回检查——Task 1/2 属口径修正，回写 PRD FR-T1 验收句与文案基线对应条目；其余不涉规格则说明。

模拟器验收通用钩子（见 RootTabView）：`-locale zh|en`、`-initialTab train|progress|plan`、`-forceOnboarding`、`-autoStartSession`、`-autoCompleteSession`。种子历史构造法：最小 canonical JSON（schemaVersion 11 + userProfile 四答 + programTemplate upper-lower/4）写入容器 `Library/Application Support/RedeData/app-data.json`，再反复「`-autoCompleteSession` 跑一场 → python 把 history 全部日期后移 2 天 → 再跑」即可得到跨 4 周真实历史（勿手搓 session）。注意 zsh 传参：脚本里从变量展开 launch 参数要用 `${=var}`。
